# 涂层配制系统文档

#### 更新时间:2026.3.31 1:24

## 1. 项目概述

本项目是一个基于PyQt5的涂层配制控制系统，通过图形界面与机械臂、泵等硬件设备交互，实现自动化配方配制流程。系统集成DeepSeek AI进行智能配方生成，并在执行前自动校验配方逻辑。

## 2. 文件结构

```
├── main.py          # 主程序入口，GUI界面与业务逻辑
├── robot.py         # 机械臂控制模块
├── vaildator.py     # JSON配方校验模块
├── main_ui.py       # PyQt5 UI代码（由Qt Designer生成）
├── main.ui          # Qt Designer界面文件
└── dobot_api.py     # 机械臂API（真机连接使用）
```

## 3. 模块功能说明

### 3.1 robot.py - 机械臂控制模块

**类名**: `RobotController`

**功能**:
- 管理机械臂的连接与控制
- 提供点位移动、夹爪控制、泵出料、搅拌等功能
- 支持模拟模式（无硬件时测试用）

**主要方法**:

| 方法 | 功能 | 参数 |
|------|------|------|
| `__init__(ip)` | 初始化控制器，设置IP地址和点位坐标 | ip: 机械臂IP地址 |
| `start(sim)` | 启动机械臂/模拟模式 | sim: True启用模拟 |
| `move_to(point_name, move_model)` | 移动机械臂到指定点位 | point_name: 点位名称, move_model: 'J'或'L' |
| `grip(state)` | 控制夹爪开合 | state: 'open'或'close' |
| `pump(pump_id, amount_ml)` | 控制泵出料 | pump_id: 泵编号(1-4), amount_ml: 出料量(ml) |
| `stir(times)` | 启动搅拌 | times: 搅拌时长(秒) |
| `execute_step(step)` | 执行单步指令 | step: 步骤字典 |

**点位定义**:

| 点位名 | 说明 |
|--------|------|
| safe | 机械臂初始安全位置 |
| arm_start | 烧杯正后方等待位 |
| arm_startforward | 前移至烧杯夹取位 |
| arm_startup | 夹取烧杯后垂直抬起位 |
| arm_stirup | 搅拌台正上方过渡位 |
| arm_stir | 搅拌台放置位 |
| arm_endup | 出料口正上方过渡位 |
| arm_end | 出料口放置位 |
| pump_1~pump_4 | 各泵出料等待位 |

### 3.2 main.py - 主程序

**核心类**:

#### LogSignaller + QtLogHandler
日志处理系统，由两个类配合完成：
- `LogSignaller`: 信号通信类，定义 `log_signal(str, str)` 信号，用于将日志从工作线程传递到主线程
- `QtLogHandler`: 继承 `logging.Handler`，将日志格式化后通过信号发送到GUI文本框，并根据级别着色显示（INFO绿色、WARNING橙色、ERROR红色）

#### ApiWorker
继承自QThread的多线程类，用于调用DeepSeek API生成配方。
- `finished` 信号: API调用成功时触发
- `error` 信号: API调用失败时触发

#### ExecuteWorker
继承自QThread的多线程类，用于执行配方步骤。
- `finished` 信号: 所有方案执行完成
- `error` 信号: 执行出错时触发
- `step_update` 信号: 每步执行时触发
- `led_update` 信号: LED状态更新

#### MainWindow
主窗口类，管理整个GUI界面。

**主要功能**:

| 功能 | 说明 |
|------|------|
| 生成配方 | 调用DeepSeek API根据用户需求生成JSON配方 |
| JSON校验 | 执行前通过 `vaildator.py` 校验配方逻辑 |
| 自动纠错 | 配方校验失败时自动请求AI修正，最多重试3次，超过则熔断保护 |
| 执行配方 | 按步骤执行机械臂、泵、搅拌等操作 |
| 模型切换 | 在普通模型(deepseek-chat)和推理模型(deepseek-reasoner)之间切换 |
| 状态显示 | 通过LED指示灯显示各模块运行状态 |
| 历史管理 | 保存对话历史，支持清空，支持多轮对话修改配方 |
| 配方保存 | 手动保存配方为本地JSON文件 |
| 日志记录 | 运行日志同时写入文件和GUI界面 |

**UI组件**:

| 组件 | 功能 |
|------|------|
| lineEdit | 用户输入需求文本 |
| start_button | 触发配方生成（显示为"解析"） |
| execute_button | 触发配方执行（显示为"开始执行"） |
| save_button | 手动保存配方（显示为"保存配方"） |
| model_button | 切换AI模型 |
| clear_button | 清空历史记录 |
| stop_button | 停止执行（功能待实现） |
| reset_button | 复位系统（功能待实现） |
| json_text | 显示生成的JSON配方（左侧） |
| run_text | 显示当前执行状态（右上） |
| runing_text | 显示运行日志（右下） |
| led_system | 系统状态指示灯 |
| led_arm | 机械臂状态指示灯 |
| led_pump1~4 | 泵1-4状态指示灯 |
| led_stir | 搅拌器状态指示灯 |

**LED状态**:
- 灰色：待机/未启动
- 黄色：运行中
- 绿色：正常完成

**自动纠错机制**:
系统内置智能纠错功能，当AI生成的配方未通过`vaildator.py`校验时：
1. 自动记录错误计数(`auto_correct_count`)
2. 将错误信息反馈给AI，请求重新生成修正配方
3. 最多重试3次，超过3次触发熔断保护，强制停止运行
4. 熔断时弹出最高警报，防止硬件损坏

### 3.3 vaildator.py - JSON配方校验模块

**功能**: 对LLM生成的配方JSON进行分层校验，防止执行时损坏硬件。

**导出函数**: `validator_formula(data) -> (bool, list)`
- 参数: `data` - 解析后的JSON字典
- 返回: `(is_ok, error_list)` - 是否通过校验 + 错误信息列表

**校验层次**:

| 层级 | 校验内容 |
|------|---------|
| 第1层 | 顶层结构完整性（task_name, requirement, formula_reasoning, plans） |
| 第2层 | plans格式与非空检查 |
| 第3层 | 每个plan内的materials校验（原料库匹配、泵号有效、总量≤400ml） |
| 第4层 | steps逻辑校验（机械爪状态、泵位空间冲突、出料量一致性、搅拌时间） |

**关键校验规则**:
- 原料必须来自原料库：环氧树脂、固化剂、稀释剂、色浆
- 泵号必须为1-4，且泵-原料对应关系与物理系统一致
- 机械爪不能重复夹紧或重复松开
- 泵出料时机械臂必须在对应泵位，且夹爪已夹持容器
- steps中的出料量必须与materials中完全一致
- 搅拌时间必须与plan中设定的stir_duration_seconds一致

**原料库定义**:

```python
all_materials = {
    1: "环氧树脂",
    2: "固化剂",
    3: "稀释剂",
    4: "色浆"
}
```

### 3.4 main_ui.py - UI界面代码

由Qt Designer生成的 `main.ui` 通过PyQt5工具自动生成。定义了所有界面组件的布局、样式和属性。不建议手动编辑此文件，应通过修改 `main.ui` 后重新生成。

### 3.5 dobot_api.py - 机械臂API

用于真机连接的Dobot机械臂API封装，提供 `DobotApiDashboard`（控制通道，端口29999）和 `DobotApiFeedBack`（反馈通道，端口30004）两个类。真机使用时取消 `robot.py` 中相关注释。

## 4. 执行流程

### 4.1 配方生成流程
1. 用户在输入框输入配制需求
2. 点击"解析"按钮
3. ApiWorker线程调用DeepSeek API（携带完整对话历史）
4. 解析返回的JSON结果，清洗markdown标记
5. 显示配方内容到左侧文本框
6. 对话历史自动追加，支持后续修改意见

### 4.2 配方执行流程
1. 用户点击"开始执行"按钮
2. 系统解析当前JSON配方
3. 调用 `validator_formula()` 进行逻辑校验
4. 校验失败：在日志框显示红色错误信息，弹窗阻断执行
5. 校验通过：
   - ExecuteWorker线程按步骤执行:
     - move: 机械臂移动（自动判断J/L运动模式）
     - grip: 夹爪控制
     - pump: 泵出料
     - stir: 搅拌
     - print: 打印标签
   - LED指示灯实时更新状态
6. 所有方案执行完成后弹窗提示

### 4.3 配方保存流程
1. 用户点击"保存配方"按钮
2. 检查配方文本框是否有有效内容
3. 提取 `task_name` 作为文件名（自动过滤非法字符）
4. 以 `{task_name}_{时间戳}.json` 格式保存到 `保存的配方/` 目录

## 5. API集成

**使用的API**: DeepSeek API（通过OpenAI SDK兼容调用）

**配置**:
- API Key: `main.py` 第18行
- Base URL: `https://api.deepseek.com`

**支持的模型**:
- `deepseek-chat` - 普通模型
- `deepseek-reasoner` - 推理模型

**系统提示词**: 包含点位定义、完整执行流程说明、原料库固定对应关系和严格的JSON输出格式要求。支持多轮对话，用户可提出修改意见，系统重新生成完整配方。

## 6. 原料库

系统固定4个泵，每种原料固定对应：

| 泵编号 | 原料 |
|--------|------|
| 1 | 环氧树脂 |
| 2 | 固化剂 |
| 3 | 稀释剂 |
| 4 | 色浆 |

总出料量上限400ml。

系统内置模拟模式，通过 `robot.start(sim=True)` 启用，用于无硬件环境下的测试。模拟模式下:
- 所有操作以 `time.sleep()` 模拟耗时
- 操作信息通过 `print()` 和 `logger` 输出
- LED状态正常更新
- 可完整测试从配方生成到执行完成的全流程

## 9. 日志系统

使用Python logging模块，采用双通道输出：

**文件日志**: 自动生成到 `运行日志/YYYY-MM-DD_log.txt`

**GUI日志**: 通过 `QtLogHandler` 实时显示在界面右下角文本框，按级别着色

**日志格式**: `时间 - 模块名 - 级别 - 消息`

**记录内容**:
- 机器人启动/连接状态
- 各步骤执行详情
- API调用结果
- 配方保存/读取操作
- 错误与警告信息

## 10. 常见问题与故障排除

- **无法启动应用**: 请确认 Python 版本为 3.8+，并且 PyQt5、openai 已正确安装。检查日志中是否有导入错误。
- **硬件无法连接**: 确认机械臂 IP 地址配置正确，网络连通性正常；若使用模拟模式，请确保 `sim=True`。
- **API 调用失败**: 检查 DeepSeek API Key 是否在 `main.py` 中正确配置，且网络可访问 `api.deepseek.com`。
- **配方校验不通过**: 查看日志框中的红色错误信息，根据提示调整输入需求重新生成配方。
- **配方未生成或输出为空**: 请确保需求文本清晰，检查 API 返回的 JSON 格式是否正确。
- **界面卡顿**: 确认所有耗时操作（API调用、机械臂控制）在工作线程中执行，未阻塞主线程。
