# 涂层配制系统文档

## 1. 项目概述

本项目是一个基于PyQt5的涂层配制控制系统，通过图形界面与机械臂、泵等硬件设备交互，实现自动化配方配制流程。

## 2. 文件结构

```
pyTest/
├── main.py      # 主程序入口，GUI界面与业务逻辑
├── robot.py     # 机械臂控制模块
├── main_ui.py   # Qt Designer生成的UI代码
└── main.ui      # Qt Designer界面文件
```

## 3. 模块功能说明

### 3.1 robot.py - 机械臂控制模块

**类名**: `RobotController`

**功能**:
- 管理机械臂的连接与控制
- 提供点位移动、夹爪控制、泵出料、搅拌等功能
- 支持模拟模式（无硬件时测试用）

**主要方法**:

| 方法 | 功能 | 参数 |
|------|------|------|
| `__init__(ip)` | 初始化控制器，设置IP地址和点位坐标 | ip: 机械臂IP地址 |
| `start(sim)` | 启动机械臂/模拟模式 | sim: True启用模拟 |
| `move_to(point_name, move_model)` | 移动机械臂到指定点位 | point_name: 点位名称, move_model: 'J'或'L' |
| `grip(state)` | 控制夹爪开合 | state: 'open'或'close' |
| `pump(pump_id, amount_ml)` | 控制泵出料 | pump_id: 泵编号(1-4), amount_ml: 出料量(ml) |
| `stir(times)` | 启动搅拌 | times: 搅拌时长(秒) |
| `execute_step(step)` | 执行单步指令 | step: 步骤字典 |

**点位定义**:

| 点位名 | 说明 |
|--------|------|
| safe | 机械臂初始安全位置 |
| arm_start | 烧杯正后方等待位 |
| arm_startforward | 前移至烧杯夹取位 |
| arm_startup | 夹取烧杯后垂直抬起位 |
| arm_stirup | 搅拌台正上方过渡位 |
| arm_stir | 搅拌台放置位 |
| arm_endup | 出料口正上方过渡位 |
| arm_end | 出料口放置位 |
| pump_1~pump_4 | 各泵出料等待位 |

### 3.2 main.py - 主程序

**核心类**:

#### LogRedirect
自定义输出重定向类，将print输出重定向到GUI文本框显示。

#### ApiWorker
继承自QThread的多线程类，用于调用DeepSeek API生成配方。
- `finished` 信号: API调用成功时触发
- `error` 信号: API调用失败时触发

#### ExecuteWorker
继承自QThread的多线程类，用于执行配方步骤。
- `finished` 信号: 所有方案执行完成
- `error` 信号: 执行出错时触发
- `step_update` 信号: 每步执行时触发
- `led_update` 信号: LED状态更新

#### MainWindow
主窗口类，管理整个GUI界面。

**主要功能**:

| 功能 | 说明 |
|------|------|
| 生成配方 | 调用DeepSeek API根据用户需求生成JSON配方 |
| 执行配方 | 按步骤执行机械臂、泵、搅拌等操作 |
| 模型切换 | 在普通模型和推理模型之间切换 |
| 状态显示 | 通过LED指示灯显示各模块运行状态 |
| 历史管理 | 保存对话历史，支持清空 |

**UI组件**:

| 组件 | 功能 |
|------|------|
| lineEdit | 用户输入需求文本 |
| start_button | 触发配方生成 |
| execute_button | 触发配方执行 |
| model_button | 切换AI模型 |
| clear_button | 清空历史记录 |
| json_text | 显示生成的JSON配方 |
| runing_text | 显示运行日志 |
| led_* | 状态指示灯(系统/机械臂/泵1-4/搅拌器) |

## 4. 执行流程

### 4.1 配方生成流程
1. 用户在输入框输入配制需求
2. 点击"生成配方"按钮
3. ApiWorker线程调用DeepSeek API
4. 解析返回的JSON结果
5. 显示配方内容

### 4.2 配方执行流程
1. 用户点击"开始执行"按钮
2. ExecuteWorker线程按步骤执行:
   - move: 机械臂移动
   - grip: 夹爪控制
   - pump: 泵出料
   - stir: 搅拌
   - print: 打印标签
3. LED指示灯实时更新状态
4. 执行完成后弹窗提示

## 5. API集成

**使用的API**: DeepSeek API

**支持的模型**:
- `deepseek-chat` - 普通模型
- `deepseek-reasoner` - 推理模型

**系统提示词**: 包含点位定义、执行流程说明和JSON输出格式要求

## 6. 硬件配置

| 设备 | 配置 |
|------|------|
| 机械臂 | IP: 192.168.5.1 |
| 泵 | 4个 (pump_1 ~ pump_4) |
| 搅拌器 | 1个 |
| 夹爪 | DO端口1 |

## 7. 模拟模式

系统内置模拟模式，通过`robot.start(sim=True)`启用，用于无硬件环境下的测试。模拟模式下:
- 所有操作记录到日志
- 执行动作以模拟速度运行
- LED状态正常更新

## 8. 日志系统

使用Python logging模块记录:
- 机器人启动/连接状态
- 各步骤执行情况
- API调用结果
- 错误信息

日志格式: `时间 - 模块名 - 级别 - 消息`
