# 涂层自动配制系统

基于 `PyQt5` 的桌面端自动配制系统，用于通过图形界面调用 DeepSeek 大模型生成配方，并联动机械臂、蠕动泵和搅拌器完成配制流程。系统内置配方安全校验、自动纠错、运行日志和本地配方归档能力，支持仿真模式与真实硬件模式。

## 项目特点

- 通过自然语言输入生成结构化 JSON 配方。
- 使用 `vaildator.py` 对配方进行执行前校验，拦截空间路径、加料顺序、物料比例和出料量错误。
- 当 AI 结果未通过校验时，自动把错误反馈给模型进行最多 3 轮纠错。
- 通过 `QThread` 异步调用模型和执行硬件动作，避免界面卡死。
- 支持串口动态刷新与切换，用于连接蠕动泵和搅拌器。
- 自动记录运行日志到 `运行日志/`，支持手动保存配方到 `保存的配方/`。
- 默认支持仿真模式，便于无硬件环境下调试流程。

## 运行环境

- Python 3.8 及以上

- DeepSeek API Key

建议安装依赖：

```bash
pip install PyQt5 openai pyserial pymodbus requests numpy
```

## 启动前配置

### 1. 配置 API Key

程序从环境变量 `DEEPSEEK_API_KEY` 读取密钥。

PowerShell 示例：

```powershell
$env:DEEPSEEK_API_KEY="你的 DeepSeek Key"
python main.py
```

### 2. 确认默认运行模式

`main.py` 中当前默认配置为：

```python
sim = True
```

这表示程序默认以仿真模式启动，不会真正驱动机械臂、泵和搅拌器。如果要连接真实硬件，需要把它改为 `False`，并确保网络与串口配置正确。

### 3. 确认机械臂 IP

当前主程序中实例化机械臂控制器使用的地址为：

```python
RobotController('192.168.201.1', simulate=sim)
```

## 快速使用

### 1. 启动程序

```bash
python main.py
```

### 2. 选择串口

程序启动后会自动扫描本机串口，并分别填充：

- 蠕动泵串口
- 搅拌器串口

如果没有检测到可用串口，界面中会显示“无可用串口”。

### 3. 生成配方

在输入框中填写需求，例如：

```text
配制一份具有防水和耐磨性能的涂层，颜色偏黑，优先考虑稳定性
```

点击“解析”后，程序会：

1. 组合系统提示词和对话历史。
2. 调用 DeepSeek 模型生成 JSON。
3. 自动执行配方校验。
4. 若校验失败，自动发起纠错。
5. 若校验通过，在左侧文本框中展示格式化后的 JSON。

### 4. 执行配方

点击“开始执行”后，程序会再次校验 JSON。只有通过校验的配方才会进入执行线程。

执行过程中包含的核心动作有：

- `move`：移动机械臂到指定点位
- `grip`：夹爪打开或闭合
- `pump`：控制指定泵按体积出料
- `stir`：启动搅拌器并维持设定时长

### 5. 保存配方

点击“保存配方”后，当前 JSON 会以如下形式保存到 `保存的配方/`：

```text
{task_name}_{时间戳}.json
```

## 当前物料库

系统当前固定使用 4 个泵，对应物料如下：

| 泵号 | 物料名称 | 比例范围 | 特殊约束 |
| --- | --- | --- | --- |
| 1 | 丙烯酸乳液 | 40% - 70% | 必须最后出料 |
| 2 | 氟化丙烯酸消光剂+消泡剂 | 1% - 10% | 若后续还有加料，则本次出料后必须搅拌 |
| 3 | 炭黑分散液 | 0% - 50% | 无 |
| 4 | 去离子水 | 0% - 30% | 无 |

单个方案总出料量不得超过 `100 ml`。

## 关键校验规则

`vaildator.py` 会重点检查以下内容：

- 顶层 JSON 字段是否完整
- `plans / materials / steps` 结构是否正确
- 泵号与物料的映射是否与系统物理配置一致
- 单个物料出料量是否大于 0，总量是否超过 100 ml
- 每种物料比例是否落在允许区间
- `add_order` 与实际 `pump` 执行顺序是否一致
- `must_be_last=True` 的物料是否最后出料
- 某些点位间是否经过安全过渡位置
- 搅拌动作是否在 `arm_stir` 点位执行
- `steps` 中累计泵出量是否与 `materials` 中声明总量一致
- 任务结束时是否释放容器并回到 `safe`

## 常见问题

### 无法生成配方

优先检查：

- 是否已设置 `DEEPSEEK_API_KEY`
- 网络是否可访问 `https://api.deepseek.com`
- 返回内容是否被模型输出为非 JSON 文本

### 无法连接硬件

优先检查：

- 机械臂 IP 是否正确
- 串口是否被其他软件占用
- 蠕动泵和搅拌器串口选择是否正确
- 当前是否仍处于 `sim = True` 仿真模式

### 配方被系统拦截

这通常说明生成结果未通过物理安全校验。请查看日志窗口中的报错详情，或重新描述需求后再次生成。
